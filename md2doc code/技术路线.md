# MD → DOCX 转换服务技术路线

**项目** BDAE ESA 报告生成系统  
**版本** 2.0  
**更新** 2026-03

---

## 1. 整体架构

```
用户输入项目信息
      ↓
Dify 工作流（多个 LLM 节点并行生成各章节）
      ↓
报告拼接节点（Code 节点，输出完整 Markdown）
      ↓
HTTP 请求节点 → pandoc-api 容器（POST /convert）
      ↓
┌─────────────────────────────────────┐
│  pandoc-api 容器                     │
│  Step 1: Pandoc  MD → DOCX          │
│          套用 reference.docx 模板    │
│  Step 2: python-docx 后处理          │
│          - 表头黑底白字              │
│          - HIGH/MED/LOW 动态染色     │
│          - 表格宽度 = 页面宽度       │
│          - 表格字体 10pt             │
└─────────────────────────────────────┘
      ↓
返回 download_url（JSON）
      ↓
代码节点解析 JSON
      ↓
End 节点输出下载链接
      ↓
用户浏览器下载 .docx
```

---

## 2. 技术选型理由

| 方案 | 优点 | 缺点 | 结论 |
|---|---|---|---|
| 纯 Pandoc | 快速、标题/表格/列表全支持 | 无法动态染色、样式控制弱 | 用于转换主体 |
| 纯 python-docx | 完全可控 | 需自写 MD 解析器，工作量大 | 仅用于后处理 |
| Pandoc + python-docx | 各做擅长的事 | 两步流程 | **采用** |
| 第三方 API | 简单 | 依赖外网、数据安全问题 | 放弃 |

---

## 3. 样式分工

### 通过 reference.docx 模板控制
- 字体（Arial / 宋体）
- 标题颜色、字号、间距
- 页面边距
- 段落行距
- 表格边框样式

### 通过 python-docx 代码控制
- 表头行背景色（Pandoc 无法区分表头行）
- HIGH / MED / LOW 基于内容的动态染色
- 表格宽度 = 页面内容宽
- 表格字体大小

## 5. 数据流详情

### Dify HTTP 请求节点配置
- 方法：POST
- URL：`http://pandoc-api:5050/convert`
- Body 类型：JSON
- Body 内容：
```json
{
  "markdown": "{{#report_merge.final_report#}}",
  "report_no": "{{#1771917764244.report_no#}}",
  "property_name": "{{#1771917764244.property_name#}}",
  "filename": "{{#1771917764244.report_no#}}_ESA_Report.docx"
}
```

### 返回值
```json
{
  "success": true,
  "download_url": "http://localhost:5050/files/ESA-001_abc12345.docx",
  "filename": "ESA-001_abc12345.docx"
}
```

### 代码节点解析
```python
import json
def main(**kwargs):
    body = kwargs.get('body', '{}')
    result = json.loads(body)
    return {
        'download_url': result.get('download_url', ''),
        'filename': result.get('filename', '')
    }
```
