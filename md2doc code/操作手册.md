## 3. 日常操作

### 3.1 重启服务
```bash
bash ~/AI_Workspace/07_Pandoc-api/start.sh
```
> ⚠️ 每次重启必须用 start.sh，不能直接用 docker run，否则挂载目录不生效

### 3.2 更新转换代码（app.py）
```bash
# 修改完 app.py 后
docker cp ~/AI_Workspace/07_Pandoc-api/app.py pandoc-api:/app/app.py
docker restart pandoc-api
```

### 3.3 更新 Word 模板（reference.docx）
1. 用 Word 打开 `~/AI_Workspace/07_Pandoc-api/reference.docx`
2. 修改样式（Heading 1 / Heading 2 / Normal 等）
3. 保存
4. 无需任何操作，下次转换自动生效（目录已挂载）

> ✅ 只改模板不需要重启容器

## 4. 样式定制指南

### 4.1 可以在 reference.docx 里修改的
在 Word 的「样式」面板中修改：
- **Heading 1** — 第一级标题（字体、颜色、字号、段前段后）
- **Heading 2** — 第二级标题
- **Heading 3** — 第三级标题
- **Normal** — 正文（字体、行距）
- 页面边距（布局 → 页边距）

> ⚠️ 修改的是样式定义，不是正文内容。正文内容由 Dify 生成。

### 4.2 需要改代码（app.py）的
在 `postprocess()` 函数中修改：

| 需求 | 修改位置 |
|---|---|
| 表头颜色 | `_set_cell_color(cell, '000000', 'FFFFFF')` 改颜色值 |
| HIGH 颜色 | `RANK_COLORS` 字典 |
| 表格字号 | `run.font.size = Pt(10)` 改数值 |
| 表格宽度 | `PAGE_WIDTH_DXA = 9639` 按实际边距计算 |

**PAGE_WIDTH_DXA 计算公式：**
```
(页面宽度cm - 左边距cm - 右边距cm) × 567
例：A4 左右各2cm → (21 - 2 - 2) × 567 = 9639
```

## 5. 常见问题

### Q: 服务启动后模板没有生效
**A:** 检查挂载是否正确：
```bash
docker exec pandoc-api ls /app/templates/
```
如果没有文件，说明启动时没用 start.sh，重新执行：
```bash
bash ~/AI_Workspace/07_Pandoc-api/start.sh
```

### Q: 转换报错 Pandoc error
**A:** 检查 Markdown 格式是否正确，常见问题：
- 表格分隔行缺少 `|---|---|`
- 代码块未闭合

### Q: 表格颜色没有变化
**A:** 检查容器里的 app.py 是否是最新版：
```bash
docker exec pandoc-api grep 'C00000' /app/app.py
```
没有输出说明旧版，重新 cp：
```bash
docker cp ~/AI_Workspace/07_Pandoc-api/app.py pandoc-api:/app/app.py
docker restart pandoc-api
```

### Q: Mac 重启后服务消失
**A:** Docker 容器默认不自动启动，每次开机后执行：
```bash
bash ~/AI_Workspace/07_Pandoc-api/start.sh
```

## 7. 接口文档

### POST /convert
转换 Markdown 为 DOCX

**请求：**
```json
{
  "markdown": "# 报告标题\n...",
  "report_no": "ESA-2024-001",
  "property_name": "项目名称",
  "filename": "ESA-2024-001_Report.docx"
}
```

**成功响应：**
```json
{
  "success": true,
  "download_url": "http://localhost:5050/files/ESA-2024-001_Report_abc12345.docx",
  "filename": "ESA-2024-001_Report_abc12345.docx"
}
```

**失败响应：**
```json
{
  "success": false,
  "error": "错误信息",
  "trace": "详细堆栈（调试用）"
}
```

### GET /health
健康检查，返回 `{"status": "ok"}`

### GET /files/{filename}
下载转换后的 DOCX 文件
